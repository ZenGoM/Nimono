# Nimono 仕様書

## 1. 概要

Nimono は、Windows デスクトップ向けの画像類似検索・グルーピングアプリケーションです。
指定したフォルダー以下の画像ファイルを再帰的に探索し、画像の内容に基づいて類似した画像をグループ化します。

## 2. 対象プラットフォーム

- Windows
- .NET 9.0
- Windows Forms アプリケーション

## 3. 主な機能

1. フォルダー選択
   - ユーザーが検索対象のフォルダーを選択できる。
   - 選択されたフォルダーは複数指定可能。
2. 画像ファイルの列挙
   - 選択したフォルダー以下を再帰的に走査して画像ファイルを収集。
   - 対象拡張子は次の通り。
     - `.jpg`, `.jpeg`, `.png`, `.bmp`, `.gif`, `.tiff`, `.tif`, `.webp`
   - 特定のシステムフォルダーはスキップする。
3. 画像ハッシュ計算
   - 各画像を 8x8 ピクセルに縮小し、グレースケール変換によって 64 ビットのハッシュを算出。
   - 並列処理でハッシュを計算し、進行状況を UI に通知する。
4. 類似画像グルーピング
   - 64 ビットハッシュのハミング距離を比較し、類似度しきい値を満たす画像を同一グループに統合。
   - 類似度しきい値は 50% 〜 100% の間で指定可能。デフォルトは 85%。
   - 2 枚以上の画像を含むグループのみ表示する。
5. 結果表示
   - グループごとにサムネイルとファイル名を表示。
   - 画像またはファイル名をクリックすると外部で開く。
   - コンテキストメニューから「開く」「フォルダーを開く」「パスをコピー」が可能。
6. キャンセル機能
   - スキャン中にキャンセルボタンを押すことで処理を中断できる。

## 4. UI 構成

### 4.1 画面レイアウト

- 上部ツールバー
  - `フォルダー選択...` ボタン
  - `スキャン開始` ボタン
  - `類似度しきい値` 入力（NumericUpDown）
  - `キャンセル` ボタン
- 中央結果領域
  - 画像グループをスクロール表示
  - 各グループにヘッダーとサムネイル一覧
- 下部ステータスバー
  - 処理状況を示すラベル
  - 進捗バー

### 4.2 状態遷移

- 初期状態
  - `フォルダー選択...` が有効
  - `スキャン開始` は無効
  - `_statusLabel` に「フォルダーを選択してください」表示
- フォルダー選択後
  - 選択フォルダーが 1 つ以上あれば `スキャン開始` を有効化
  - ステータスに選択フォルダー数を表示
- スキャン中
  - 入力操作を無効化して処理を保護
  - 進捗バーを更新
  - `キャンセル` が有効化
- スキャン完了 / キャンセル / エラー
  - UI を再び操作可能に戻す
  - 結果またはメッセージを表示

## 5. 処理の流れ

1. ユーザーがフォルダーを選択
2. `ImageGrouper.EnumerateImages` で画像ファイル一覧を収集
3. `ImageGrouper.ComputeHashes` で画像ハッシュを並列計算
4. `ImageGrouper.Group` で類似度に基づきグループ化
5. `MainForm.RenderGroups` で結果を表示

## 6. 主要クラスと責務

### 6.1 `MainForm`

- UI の初期化と構築
- ユーザー操作のハンドリング
- スキャン処理の制御とキャンセル管理
- グループ結果の描画
- サムネイル生成と外部アプリ起動

### 6.2 `ImageGrouper`

- 画像ファイルの再帰列挙
- 並列ハッシュ計算
- 類似画像のグループ化
- 進行状況報告

### 6.3 `ImageHasher`

- 8x8 画像リサイズ
- グレースケール平均による 64 ビットハッシュ生成
- ハミング距離計算
- 類似度計算

### 6.4 `Program`

- Windows Forms アプリケーションのエントリポイント

## 7. 技術詳細

### 7.1 画像ハッシュ

- 画像を 8x8 に縮小
- 各ピクセルをグレースケール化
- 平均輝度以上のピクセルを 1、未満を 0 にして 64 ビットハッシュを生成
- ハミング距離 = `BitOperations.PopCount(h1 ^ h2)`
- 類似度 = `1.0 - distance / 64`

### 7.2 グループ判定

- 指定された類似度しきい値 `T` に対し、最大距離を `floor(64 * (1 - T))` で計算
- 2 つの画像ハッシュ間の距離がこの閾値以下なら同じグループとみなす
- Union-Find で連結成分を統合
- サイズ 2 以上の成分のみを結果として返す

### 7.3 対象ファイル・ディレクトリの扱い

- 対象画像拡張子
  - `jpg`, `jpeg`, `png`, `bmp`, `gif`, `tiff`, `tif`, `webp`
- 走査時にスキップするディレクトリ名
  - `windows`, `program files`, `program files (x86)`, `$recycle.bin`, `system volume information`
- `UnauthorizedAccessException` / `IOException` は無視して継続

## 8. 例外とキャンセル

- 画像読み込み失敗時は個別ファイルをスキップ
- 全体処理中の例外はメッセージとして表示
- 操作がキャンセルされた場合、`OperationCanceledException` をキャッチして「キャンセルされました」と表示
- フォーム終了時にもキャンセル要求を送信し、リソースを解放

## 9. ユーザー操作

- スキャン対象フォルダーを選択
- 類似度のしきい値を設定
- 「スキャン開始」で検索を実行
- 必要なら「キャンセル」で中断
- 検索結果のサムネイルをクリックして画像を開く
- コンテキストメニューでフォルダー表示やパスコピー

## 10. 制限と備考

- 画像比較はあくまで 8x8 の単純なハッシュに基づくため、精度は完全ではない。
- 大量の画像では O(n^2) 比較が発生するため、処理時間が増加する。
- サムネイル生成は読み込み失敗時に空白で表示される。
- 選択したフォルダーは UI の外部に保持され、ウィンドウ内でのみ使用される。
