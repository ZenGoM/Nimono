# Nimono 仕様書 — メイン画面

## 1. 概要

Nimono は Windows デスクトップ向けの画像類似検索・グルーピングアプリケーションです。
指定したフォルダー以下の画像ファイルを再帰的に探索し、画像の内容に基づいて類似した画像をグループ化します。
グループ化結果をサムネイル一覧で表示し、類似画像の確認・比較を支援します。

## 4. ツールバー

| コントロール | 動作 |
|------------|------|
| `フォルダー選択...` | `SearchFolderDialog` を開き、スキャン対象フォルダーを追加・削除する |
| `スキャン開始` | スキャンを開始する。フォルダー未選択時は無効 |
| `計算方式` | ドロップダウン。`pHash（高速）` / `DINOv2 CPU` / `DINOv2 GPU (DirectML)` から選択 |
| `類似度しきい値` | 50〜100 の整数（%）を入力。デフォルト 85。変更時に設定を即保存 |
| `キャッシュクリア` | キャッシュ DB を削除する。ボタン名に現在のキャッシュ容量を動的に表示する（例: `キャッシュクリア (12.3 MB)`）。確認ダイアログあり。スキャン中は無効 |
| `キャンセル` | スキャン中のみ有効。処理を中断する |

## 5. スキャン処理

### 5.1 処理ステップ（pHash モード）

| ステップ | 処理内容 | 担当クラス |
|---------|---------|-----------|
| 1 | フォルダーを再帰走査して画像ファイルを列挙 | `ImageGrouper.EnumerateImages` |
| 2 | キャッシュ DB からメモリに読み込み | `CacheManager.LoadCacheAsync` |
| 3 | 各画像の 64bit pHash を並列計算（キャッシュヒット時はスキップ） | `ImageGrouper.ComputeHashes` |
| 4 | ハミング距離でグループ化（Union-Find） | `ImageGrouper.Group` |
| 5 | 新規・変更分のキャッシュを DB に保存 | `CacheManager.SaveCacheAsync` |
| 6 | 結果をメイン画面に表示 | `MainForm.RenderGroups` |

### 5.2 処理ステップ（DINOv2 モード）

| ステップ | 処理内容 | 担当クラス |
|---------|---------|-----------|
| 1 | フォルダーを再帰走査して画像ファイルを列挙 | `ImageGrouper.EnumerateImages` |
| 2 | キャッシュ DB からメモリに読み込み | `CacheManager.LoadCacheAsync` |
| 3 | 各画像の DINOv2 埋め込みベクトルを並列計算（キャッシュヒット時はスキップ） | `ImageGrouper.ComputeEmbeddings` |
| 4 | コサイン類似度でグループ化（Union-Find） | `ImageGrouper.GroupByEmbedding` |
| 5 | 新規・変更分のキャッシュを DB に保存 | `CacheManager.SaveCacheAsync` |
| 6 | 結果をメイン画面に表示 | `MainForm.RenderGroups` |

### 5.4 スキップするディレクトリ

## 6. キャッシュシステム

### 6.1 概要

スキャン結果（pHash / DINOv2 埋め込みベクトル）を SQLite データベースにキャッシュし、再スキャン時に計算をスキップする。キャッシュはファイルサイズと最終更新日時をキーとして有効性を判定する。

### 6.2 キャッシュ DB

- 保存先: `%LOCALAPPDATA%\Nimono\nimono_cache.db`
- 形式: SQLite
- 管理クラス: `CacheManager`

### 6.3 キャッシュエントリ

| フィールド | 型 | 内容 |
|-----------|-----|------|
| `Path` | `string` | ファイルのフルパス（主キー） |
| `FileSize` | `long` | ファイルサイズ（バイト） |
| `LastWriteTimeTicks` | `long` | 最終更新日時（Ticks） |
| `PHash` | `ulong?` | pHash 値（未計算なら null） |
| `Embedding` | `float[]?` | DINOv2 埋め込みベクトル（未計算なら null） |

## 7. グループ表示

### 7.1 グループの定義

- **pHash モード**: ハミング距離 ≤ `floor(64 × (1 − T))` の画像を Union-Find で統合
- **DINOv2 モード**: コサイン類似度 ≥ T の画像を Union-Find で統合
- 2 枚以上の画像を含むグループのみ表示
- グループは画像枚数の多い順に表示

### 7.2 グループヘッダー

- 背景色: `RGB(60, 90, 140)`（青系）/ 高さ 28px
- テキスト: `グループ {ID} — {枚数} 枚`
- 右端に「比較」ボタン（クリックで比較画面を開く → [COMPARE_SPEC.md](COMPARE_SPEC.md)）

### 7.3 グループ内の画像順序

- 基準画像（最も高い類似度 = 100%）が先頭
- 以降は基準画像との類似度の降順
- サムネイルをクリックすると、クリックした画像が新たな基準となり再ソート

### 7.4 仮想スクロール（遅延パネル生成）

大量グループ（例: 12,000+ グループ）が存在する場合のパフォーマンスを確保するため、仮想スクロールと遅延パネル生成を採用しています。

- `RenderGroups` ではパネルを生成せず、各グループの高さを**推定**して絶対位置（`_groupTops`）を記録
- `AutoScrollMinSize` を推定総高さに設定してスクロールバーを制御
- 表示領域 ± 1 viewport 分のグループが可視域に入った時点で初めて `BuildGroupPanel` を呼び出し、パネル・サムネイルを生成する
- 実測高さが推定と異なる場合は、後続グループの位置を自動的に再計算する
- 表示領域から 3 viewport 以上離れたパネルは破棄してメモリを解放する
- スクロールイベント（`WM_VSCROLL` / `WM_MOUSEWHEEL` / `WM_KEYDOWN`）で表示グループを更新

## 9. 画像ハッシュアルゴリズム（pHash）

1. 画像を 8×8 ピクセルに縮小（HighQualityBicubic）
2. グレースケール変換: `0.299R + 0.587G + 0.114B`
3. 64 ピクセルの平均輝度を計算
4. 各ピクセルが平均以上なら 1、未満なら 0 の 64bit ハッシュを生成

```
ハミング距離  = BitOperations.PopCount(hash1 ^ hash2)
類似度        = 1.0 − distance / 64.0
最大距離閾値  = floor(64 × (1 − T))   ※ T = 類似度しきい値
```

## 10. DINOv2 アルゴリズム

### 10.1 概要

DINOv2 は Meta が開発した自己教師あり学習モデルで、画像の意味的な特徴を捉えた埋め込みベクトルを生成する。pHash よりも高精度な類似画像検出が可能。

### 10.2 処理フロー

1. 画像を 224×224 ピクセルにリサイズ（HighQualityBicubic）
2. ImageNet の平均・標準偏差で正規化（BGR → RGB 変換込み）
3. ONNX Runtime でモデルを実行し、CLS トークンの埋め込みベクトルを取得
4. L2 正規化を適用

### 10.3 類似度計算

```
コサイン類似度 = dot(a, b)   ※ 両ベクトルは L2 正規化済み
```

### 10.4 モデルファイル

- 形式: ONNX（`.onnx`）
- 自動検出: 実行ファイルと同じディレクトリの `model.onnx` を自動読み込み
- 実行プロバイダー: CPU または DirectML（GPU）

### 10.5 DirectML（GPU）動作仕様

DINOv2 の埋め込み計算に DirectML を利用する場合の動作仕様は以下の通りです。

- **ONNX Runtime 実行プロバイダー**:
  - 計算方式で「DINOv2 GPU (DirectML)」が選択された場合、ONNX Runtime の `AppendExecutionProvider_DML(0)` を呼び出し、DirectX 12 互換 GPU で推論処理を実行する。
  - DirectML は NVIDIA / AMD / Intel / 統合 GPU を問わず DX12 対応であれば動作し、ベンダー固有ランタイム（CUDA Toolkit / cuDNN）は不要。
  - `GraphOptimizationLevel` は `ORT_ENABLE_BASIC` を使用（DirectML EP との互換性確保）。
- **起動時スモークテスト**:
  - セッション作成直後に全ゼロのダミーテンソル `[1,3,224,224]` で 1 回 `Run()` を実行し、Run 時に発生する DirectML エラー（ドライバ不整合・カーネル不在など）を検出する。
  - 検出した場合はセッションを破棄し、ユーザーに「DirectML (GPU) の初期化に失敗しました」とエラーダイアログを表示してスキャンを中止する（自動 CPU フォールバックは行わない。ユーザーが手動で「DINOv2 CPU」に切り替える）。

### 10.6 DirectML 環境の要件

- **OS**: Windows 10 1709 (RS3) 以降、または Windows 11
- **GPU**: DirectX 12 対応 GPU
- **グラフィックドライバー**: 各 GPU ベンダーの最新ドライバー（DirectML は WDDM 2.x 系に対応）
- **追加インストール**: 不要（DirectML ランタイムは Windows に同梱、追加 DLL は NuGet パッケージ `Microsoft.ML.OnnxRuntime.DirectML` に含まれる）

### 10.7 DirectML の動作確認方法

PowerShell で GPU が DirectX 12 をサポートしているか確認します。

```powershell
dxdiag /t dxdiag.txt
# 出力ファイルの "Feature Levels" に "12_0" 以上が含まれることを確認
```

または、Nimono を起動して計算方式に「DINOv2 GPU (DirectML)」を選び、小さなフォルダーでスキャンを実行することで、起動時のスモークテストが成功するかどうかが判定できる。

## 13. 既知の制約

- pHash は 8×8 の粗いハッシュのため、近似的な一致のみ検出。精度は完全ではない。
- グループ化は O(n²) の全ペア比較のため、画像数が多いほど処理時間が増加する。
- サムネイル読み込み失敗時は灰色の空欄で表示される。
- DINOv2 モードでは ONNX モデルファイルが必要。モデル未設定時はスキャン開始時にメッセージを表示する。
- 仮想スクロールにより Win32 の 32,767px 座標上限を回避している。可視域外のパネルは破棄されるため、スクロール時にサムネイル再生成コストが発生する。
