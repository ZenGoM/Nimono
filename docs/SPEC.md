# Nimono 仕様書 — メイン画面

## 1. 概要

Nimono は Windows デスクトップ向けの画像類似検索・グルーピングアプリケーションです。
指定したフォルダー以下の画像ファイルを再帰的に探索し、画像の内容に基づいて類似した画像をグループ化します。
グループ化結果をサムネイル一覧で表示し、類似画像の確認・比較を支援します。

## 4. ツールバー

| コントロール | 動作 |
|------------|------|
| `フォルダー選択...` | `SearchFolderDialog` を開き、スキャン対象フォルダーを追加・削除する |

## 5. スキャン処理

### 5.1 処理ステップ

| ステップ | 処理内容 | 担当クラス |
|---------|---------|-----------|
| 1 | フォルダーを再帰走査して画像ファイルを列挙 | `ImageGrouper.EnumerateImages` |
| 2 | 各画像の 64bit pHash を並列計算 | `ImageGrouper.ComputeHashes` |
| 3 | ハミング距離でグループ化（Union-Find） | `ImageGrouper.Group` |
| 4 | 結果をメイン画面に表示 | `MainForm.RenderGroups` |

### 5.2 対象拡張子

`.jpg` `.jpeg` `.png` `.bmp` `.gif` `.tiff` `.tif` `.webp`

### 5.3 スキップするディレクトリ

`windows` / `program files` / `program files (x86)` / `$recycle.bin` / `system volume information`

### 5.4 UI の状態変化

| 状態 | フォルダー選択 | スキャン開始 | キャンセル | 説明 |
|------|:---:|:---:|:---:|------|
| 初期 | 有効 | 無効 | 無効 | 「フォルダーを選択してください」 |
| フォルダー選択済み | 有効 | 有効 | 無効 | 「N フォルダー選択中」 |
| スキャン中 | 無効 | 無効 | 有効 | 進捗バー更新 |
| 完了 | 有効 | 有効 | 無効 | グループ数・枚数を表示 |

| エラー | 有効 | 有効 | 無効 | エラーメッセージを表示 |

## 6. グループ表示

### 6.1 グループの定義

- 類似度しきい値以上（ハミング距離 ≤ `floor(64 × (1 − T))`）の画像を Union-Find で統合
- 2 枚以上の画像を含むグループのみ表示
- グループは画像枚数の多い順に表示

### 6.2 グループヘッダー

- 背景色: `RGB(60, 90, 140)`（青系）/ 高さ 28px
- テキスト: `グループ {ID} — {枚数} 枚`
- 右端に「比較」ボタン（クリックで比較画面を開く → [COMPARE_SPEC.md](COMPARE_SPEC.md)）

### 6.3 グループ内の画像順序

- 基準画像（最も高い類似度 = 100%）が先頭
- 以降は基準画像との類似度の降順
- サムネイルをクリックすると、クリックした画像が新たな基準となり再ソート

### 6.4 仮想スクロール

大量グループ（例: 451 グループ × 約 300px ≒ 137,000px）が存在する場合、Win32 の子ウィンドウ座標上限（32,767px）を超えてしまうため、仮想スクロールを採用しています。

- 全グループのパネルを事前生成し、絶対位置（`_groupTops`）を記録
- `AutoScrollMinSize` を総高さに設定してスクロールバーを制御
- 表示領域 ± 1 viewport 分のグループのみ `Controls` に追加（物理座標 = 絶対位置 − スクロール位置）
- スクロールイベント（`WM_VSCROLL` / `WM_MOUSEWHEEL` / `WM_KEYDOWN`）で表示グループを更新

## 8. 画像ハッシュアルゴリズム（pHash）

1. 画像を 8×8 ピクセルに縮小（HighQualityBicubic）
2. グレースケール変換: `0.299R + 0.587G + 0.114B`
3. 64 ピクセルの平均輝度を計算
4. 各ピクセルが平均以上なら 1、未満なら 0 の 64bit ハッシュを生成

```
ハミング距離  = BitOperations.PopCount(hash1 ^ hash2)
類似度        = 1.0 − distance / 64.0
最大距離閾値  = floor(64 × (1 − T))   ※ T = 類似度しきい値
```

## 9. 設定の永続化

保存先: `%LOCALAPPDATA%\Nimono\settings.json`

| プロパティ | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| `SearchFolders` | `string[]` | `[]` | スキャン対象フォルダー一覧 |
| `SimilarityThreshold` | `int` | `85` | 類似度しきい値（%） |
| `WindowWidth` | `int` | `1100` | メイン画面の幅（px） |
| `WindowHeight` | `int` | `750` | メイン画面の高さ（px） |
| `CompareFormWidth` | `int` | `900` | 比較画面の幅（px） |
| `CompareFormHeight` | `int` | `700` | 比較画面の高さ（px） |
| `CompareSplitterDistance` | `int` | `0` | 比較画面の上下境目位置（px、0 = 自動） |

保存タイミング:
- フォルダー選択変更時
- しきい値変更時
- ウィンドウサイズ変更完了時（`ResizeEnd` イベント）

## 11. 既知の制約

- pHash は 8×8 の粗いハッシュのため、近似的な一致のみ検出。精度は完全ではない。
- グループ化は O(n²) の全ペア比較のため、画像数が多いほど処理時間が増加する。
- サムネイル読み込み失敗時は灰色の空欄で表示される。
- 仮想スクロールにより Win32 の 32,767px 座標上限を回避しているが、表示領域外のパネルは Controls から除去されるため、スクロール時に追加コストが発生する。
