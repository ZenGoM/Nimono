# Nimono 仕様書 — 比較画面

## 1. 概要

比較画面（`CompareForm`）は、メイン画面のグループヘッダーにある「比較」ボタンから開く補助ウィンドウです。
同一グループ内の 2 枚の画像を並べて表示し、細部の差異を視覚的に確認することを目的としています。

---

## 2. 画面レイアウト

```
┌─────────────────────────────────────────────────────────┐
│          比較 — グループ {ID}（{N} 枚）                   │ ← タイトルバー
├──────────────────────────┬──────────────────────────────┤
│   基準（固定）            │   比較対象                   │ ← ロールラベル（各 22px）
│                          │                              │
│                          │                              │
│        [左画像]           │        [右画像]              │ ← PictureBox（Zoom）
│                          │                              │
│                          │                              │
│  ファイル名.jpg           │  ファイル名2.png              │
│  【基準】 JPG · 1.2 MB   │  類似度: 94%  PNG · 1.0 MB  │ ← 情報ラベル（各 64px）
│  1920×1080  2024/01/15   │  1920×1080  2024/01/20      │
│  /path/to/file.jpg       │  /path/to/file2.png          │
├══════════════════════════════════════════════════════════╡ ← スプリッター（6px、ドラッグ可）
│ ファイル名           形式  サイズ   解像度    日付    類似度 │
│ ファイル名.jpg [基準] JPG  1.2 MB  1920×1080 24/01/15 100% │
│ ファイル名2.png       PNG  1.0 MB  1920×1080 24/01/20  94% │ ← 選択行（右パネルに表示）
│ ...                                                      │
└─────────────────────────────────────────────────────────┘
```

左右の比率は常に **50:50** で固定です（スプリッターなし）。

---

## 3. 上部 — 画像表示エリア

### 3.1 全体構成

- `TableLayoutPanel`（2 列 50%:50%）で左右に分割
- 両パネルの背景色: `RGB(235, 235, 240)`
- パネル間の 2px セパレーター: `RGB(160, 160, 170)`

### 3.2 左パネル（基準）

| 要素 | 内容 |
|------|------|
| ロールラベル（22px） | 「基準（固定）」/ 背景 `RGB(60, 90, 140)`（青）|
| PictureBox | グループの先頭画像（`Paths[0]`）を固定表示 |
| 情報ラベル（64px） | ファイル名 / 役割・形式・サイズ・解像度・日付 / フルパス |

左パネルの表示画像は変更できません。

### 3.3 右パネル（比較対象）

| 要素 | 内容 |
|------|------|
| ロールラベル（22px） | 「比較対象」/ 背景 `RGB(60, 120, 60)`（緑）|
| PictureBox | ファイルリストで選択中の画像を表示 |
| 情報ラベル（64px） | ファイル名 / 類似度・形式・サイズ・解像度・日付 / フルパス |

初期選択: グループの 2 番目の画像（`Paths[1]`）。

### 3.4 情報ラベルの書式

```
{ファイル名}
{役割}  {形式} · {ファイルサイズ}  {解像度}  {日付}
{フルパス}
```

- 基準画像の役割: `【基準】`
- 比較対象の役割: `類似度: {XX}%`
- 解像度はオリジナル画像の実寸（読み込み縮小前）を表示

### 3.5 PictureBox の操作

| 操作 | 動作 |
|------|------|
| ダブルクリック | 既定のアプリで画像を開く |
| 右クリック | コンテキストメニューを表示（後述） |
| ホバー | ツールチップにフルパスを表示 |

---

## 4. 下部 — ファイルリスト

### 4.1 表示列

| 列 | 幅（px） | 内容 |
|----|---------|------|
| ファイル名 | 220 | ファイル名。基準画像には `[基準]` を付記 |
| 形式 | 55 | 拡張子の大文字表記（例: `JPG`） |
| サイズ | 80 | ファイルサイズ（MB / KB / B） |
| 解像度 | 100 | `幅×高さ`（例: `1920×1080`） |
| 日付 | 90 | 更新日時（`yyyy/MM/dd`） |
| 類似度 | 70 | 基準画像との類似度（例: `94%`） |

### 4.2 選択動作

- シングルクリックで行を選択 → 右パネルの画像・情報ラベルが更新される
- ダブルクリック → 既定のアプリで画像を開く

### 4.3 コンテキストメニュー（右クリック）

| メニュー項目 | 動作 |
|------------|------|
| 開く | 既定のアプリで選択行の画像を開く |
| フォルダーを開く | Explorer でファイルを選択した状態で開く |
| パスをコピー | クリップボードにフルパスをコピー |

---

## 5. スプリッター（上下境目）

| 項目 | 仕様 |
|------|------|
| 方向 | 水平（上下分割） |
| 幅 | 6px |
| デフォルト位置 | ウィンドウ高さの 65% |
| 操作 | マウスドラッグで位置を変更可能 |
| 記憶 | 位置を `settings.json` に保存し、次回起動時に復元 |

---

## 6. 画像読み込み

- 最大 2,000×2,000px に縮小して読み込む（メモリ節約）
- それ以下の画像はオリジナルサイズで読み込む
- 読み込み失敗時は PictureBox を空（灰色）で表示
- 解像度情報（情報ラベル・ファイルリスト）はオリジナルのヘッダーから読み取り、PictureBox の縮小サイズとは独立して表示

---

## 7. 設定の永続化

| プロパティ（`AppSettings`） | 保存タイミング | 内容 |
|---------------------------|-------------|------|
| `CompareFormWidth` | `ResizeEnd` | ウィンドウ幅（px） |
| `CompareFormHeight` | `ResizeEnd` | ウィンドウ高さ（px） |
| `CompareSplitterDistance` | `SplitterMoved` | スプリッター位置（px） |

保存先: `%LOCALAPPDATA%\Nimono\settings.json`（メイン画面と共有）

---

## 8. ウィンドウ動作

| 項目 | 仕様 |
|------|------|
| 起動位置 | メイン画面の中央（`CenterParent`） |
| 最小サイズ | 600×400px |
| デフォルトサイズ | 900×700px（設定がない場合） |
| モード | モーダル（`ShowDialog`）。メイン画面は操作不可 |
| 終了時 | 読み込んだ Bitmap を破棄してリソースを解放 |

---

## 9. クラス構成

```
CompareForm
├── _group: ImageGroup          グループデータ（Paths / Similarities / Hashes）
├── _settings: AppSettings      設定（サイズ・スプリッター位置の読み書き）
├── _sizeCache: Dictionary      PopulateList で取得した解像度文字列のキャッシュ
├── _mainSplitter: SplitContainer  上下分割
├── _leftPic / _rightPic        各 PictureBox
├── _leftInfo / _rightInfo      各情報ラベル
├── _fileList: ListView         ファイルリスト
├── _leftBitmap / _rightBitmap  表示中の Bitmap（Dispose 管理）
└── _suppressEvent: bool        プログラム選択時のイベント二重発火防止フラグ
```

---

## 10. 制約・注意事項

- 左パネルは常にグループ先頭（基準）画像で固定。メイン画面でサムネイルクリックにより基準が変更されている場合は、その時点の `group.Paths[0]` が使われる。
- 比較画面を開いたままメイン画面で基準変更を行っても、比較画面には反映されない（再度「比較」ボタンを押す必要がある）。
- 解像度列はファイルヘッダーの読み取りを伴うため、ファイル数が多いグループでは画面表示に若干時間がかかる場合がある。
