# Nimono 仕様書 — 比較画面

## 1. 概要

比較画面（`CompareForm`）は、メイン画面のグループヘッダーにある「比較」ボタンから開く補助ウィンドウです。
同一グループ内の 2 枚の画像を並べて表示し、細部の差異を視覚的に確認することを目的としています。

---

## 2. 画面レイアウト

```
┌─────────────────────────────────────────────────────────┐
│          比較 — グループ {ID}（{N} 枚）                   │ ← タイトルバー
├──────────────────────────┬──────────────────────────────┤
│   基準（固定）            │   比較対象                   │ ← ロールラベル（各 22px）
│                          │                              │
│                          │                              │
│        [左画像]           │        [右画像]              │ ← PictureBox（Zoom）
│                          │                              │
│                          │                              │
│  ファイル名.jpg           │  ファイル名2.png              │
│  【基準】                 │  類似度: 94%                 │ ← 情報ラベル（各 80px）
│  JPG · 1.2 MB  1920×1080 │  PNG · 1.0 MB  1920×1080    │
│  2024/01/15               │  2024/01/20                  │
│  C:\path\to\dir           │  C:\path\to\dir2             │
├══════════════════════════════════════════════════════════╡ ← スプリッター（6px、ドラッグ可）
│ 類似度  ファイル名       形式  サイズ   解像度    日付   パス    │
│ 【基準】ファイル名.jpg   JPG  1.2 MB  1920×1080 24/01/15 ...  │
│  94%   ファイル名2.png   PNG  1.0 MB  1920×1080 24/01/20 ...  │ ← 選択行（右パネルに表示）
│ ...                                                      │
└─────────────────────────────────────────────────────────┘
```

左右の比率は常に **50:50** で固定です（スプリッターなし）。

---

## 3. 上部 — 画像表示エリア

### 3.1 全体構成

- `TableLayoutPanel`（2 列 50%:50%）で左右に分割
- 両パネルの背景色: `RGB(235, 235, 240)`
- パネル間の 2px セパレーター: `RGB(160, 160, 170)`

### 3.2 左パネル（基準）

| 要素 | 内容 |
|------|------|
| ロールラベル（22px） | 「基準（固定）」/ 背景 `RGB(60, 90, 140)`（青）|
| PictureBox | グループの先頭画像（`Paths[0]`）を固定表示 |
| 情報ラベル（80px） | ファイル名 / 役割 / 形式・サイズ・解像度・日付 / ディレクトリパス |
| 削除ボタン | 「削除」（右上にオーバーレイ） |

左パネルの表示画像は変更できません。

### 3.3 右パネル（比較対象）

| 要素 | 内容 |
|------|------|
| ロールラベル（22px） | 「比較対象」/ 背景 `RGB(60, 120, 60)`（緑）|
| PictureBox | ファイルリストで選択中の画像を表示 |
| 情報ラベル（80px） | ファイル名 / 類似度 / 形式・サイズ・解像度・日付 / ディレクトリパス |
| 削除ボタン | 「削除」（右上にオーバーレイ） |

初期選択: グループの 2 番目の画像（`Paths[1]`）。

### 3.4 情報ラベルの書式

```
{ファイル名}
{役割}
{形式} · {ファイルサイズ}  {解像度}  {日付}
{ディレクトリパス}
```

- 基準画像の役割: `【基準】`
- 比較対象の役割: `類似度: {XX}%`
- 役割とファイル形式の間は改行（計 4 行）
- ラベル高さ: 80px
- 解像度はオリジナル画像の実寸（読み込み縮小前）を表示
- **ディレクトリパス**: ファイル名を除いたパス部分のみ表示（`Path.GetDirectoryName`）

### 3.5 差分ハイライト

情報ラベルは `RichTextBox`（読み取り専用・枠なし）で実装されており、基準と比較対象の各フィールドを比較して **異なる部分のみ赤字**で表示する。

| フィールド | 比較方法 | 差分単位 |
|-----------|---------|---------| 
| ファイル名 | 文字列比較 | 文字単位（共通プレフィックス・サフィックスを通常色、中間差分を赤字） |
| 形式 | 拡張子（大文字） | フィールド単位 |
| サイズ | ファイルサイズ（バイト数） | フィールド単位 |
| 解像度 | 解像度文字列（`幅×高さ`） | フィールド単位 |
| 日付 | 更新日時（`yyyy/MM/dd`） | フィールド単位 |
| ディレクトリパス | パス文字列 | 文字単位（共通プレフィックス・サフィックスを通常色、中間差分を赤字） |

- 役割行（`【基準】` / `類似度: XX%`）は常に通常色（比較しない）。ただし類似度 100% 未満の場合は赤字
- フィールド間の区切り文字（` · `、スペース）は常に通常色
- 右パネルの選択が変わると両パネルの情報ラベルを同時に更新する

### 3.6 画像パネルの操作（ZoomableImagePanel）

| 操作 | 動作 |
|------|------|
| マウスホイール | ズームイン/アウト（カーソル位置を中心に拡縮） |
| 左ドラッグ | パン（画像を移動） |
| ダブルクリック | ビューをリセット（ズーム 1x・中央表示） |
| 右クリック | コンテキストメニューを表示 |
| ホバー | ツールチップにフルパスを表示 |

#### 3.6.1 ズーム・パンの仕様

- ズーム範囲: 0.05x〜50x（フィットサイズ基準）
- カーソル位置を固定したままズームが行われる
- 左右パネルのズーム倍率とパン量は **常に同期**。片方を操作するともう一方も連動する
- 右パネルの画像をリスト選択で切り替えても、ズーム・パン位置はリセットされない（同じ位置を比較し続けられる）
- 「比較」ボタンで新規に開いた際はズーム 1x（全体表示）から開始

#### 3.6.2 同期の仕組み

ズームは「フィットサイズに対する倍率」、パンは「スクリーン座標（px）」で管理する。
両パネルが同一サイズ（50:50）のため、同じ倍率・同じパン量を適用すると、同じ画像領域が同じスケールで表示される。

#### 3.6.3 コンテキストメニュー

| メニュー項目 | 動作 |
|------------|------|
| 開く | 既定のアプリで画像を開く |
| 回転 | 画像を 90° 右回転する（設定に保存） |
| フォルダーを開く | Explorer でファイルを選択した状態で開く |
| パスをコピー | クリップボードにフルパスをコピー |

## 4. 下部 — ファイルリスト

### 4.1 表示列

| # | 列 | 幅（px） | 内容 |
|---|----|---------|------|
| 1 | 類似度 | 70 | 基準画像には `【基準】`、それ以外は類似度（例: `94%`） |
| 2 | ファイル名 | 200 | ファイル名のみ（付記なし） |
| 3 | 形式 | 55 | 拡張子の大文字表記（例: `JPG`） |
| 4 | サイズ | 80 | ファイルサイズ（MB / KB / B） |
| 5 | 解像度 | 100 | `幅×高さ`（例: `1920×1080`） |
| 6 | 日付 | 90 | 更新日時（`yyyy/MM/dd`） |
| 7 | パス | 300 | ディレクトリパス（ファイル名を除く） |

列幅は変更可能で、`settings.json` に保存される。

### 4.2 選択動作

| 操作 | 動作 |
|------|------|
| シングルクリック | 行を選択 → 右パネルの画像・情報ラベルが更新される |
| ダブルクリック | クリックした画像を基準に入れ替える（基準行は無視） |
| 基準行のクリック | 選択を元の比較対象行に戻す（基準行は選択不可） |

### 4.3 コンテキストメニュー（右クリック）

| メニュー項目 | 動作 |
|------------|------|
| 開く | 既定のアプリで選択行の画像を開く |
| フォルダーを開く | Explorer でファイルを選択した状態で開く |
| パスをコピー | クリップボードにフルパスをコピー |

---

## 5. 基準入れ替え

ファイルリストの行をダブルクリックすると、その画像が新しい基準画像になる。

1. クリックされた画像を `Paths[0]` に移動
2. 全画像の類似度を新しい基準に対して再計算（DINOv2 ならコサイン類似度、pHash ならハミング距離）
3. 類似度の降順で再ソート
4. ファイルリストを再構築
5. 左パネルに新しい基準画像を表示
6. 右パネルには元の基準画像を選択

---

## 6. 削除機能

### 6.1 操作

左パネル・右パネルそれぞれの「削除」ボタンで、表示中の画像をゴミ箱に移動する。

### 6.2 確認ダイアログ

- 初回はタスクダイアログで確認（「次回以降確認しない」チェックボックスあり）
- チェックを入れると以降はセッション中の全削除が即実行される

### 6.3 削除後の動作

| 削除対象 | 動作 |
|---------|------|
| 右パネル（比較対象） | リストから除去し、次の画像を選択 |
| 左パネル（基準） | 2 番目の画像を新しい基準とし、類似度を再計算 |
| グループが 2 枚未満 | 比較画面を自動的に閉じる |

### 6.4 メイン画面への反映

比較画面を閉じた際、削除や基準変更の結果がメイン画面のグループに反映される。グループが消滅した場合はメイン画面からも除去される。

## 7. キーボードナビゲーション

| キー | 動作 |
|------|------|
| Home | ファイルリストの先頭（2 番目の画像）を選択 |
| End | ファイルリストの末尾を選択 |
| PageUp | 1 ページ分上にスクロール |
| PageDown | 1 ページ分下にスクロール |

ページサイズはリストの表示行数から自動計算される。

## 8. スプリッター（上下境目）

| 項目 | 仕様 |
|------|------|
| 方向 | 水平（上下分割） |
| 幅 | 6px |
| デフォルト位置 | ウィンドウ高さの 65% |
| 操作 | マウスドラッグで位置を変更可能 |
| 記憶 | 位置を `settings.json` に保存し、次回起動時に復元 |

## 9. 画像読み込み

- 最大 2,000×2,000px に縮小して読み込む（メモリ節約）
- それ以下の画像はオリジナルサイズで読み込む
- 読み込み失敗時は PictureBox を空（灰色）で表示
- 解像度情報（情報ラベル・ファイルリスト）はオリジナルのヘッダーから読み取り、PictureBox の縮小サイズとは独立して表示
- 保存済みの回転情報がある場合、読み込み時に自動的に回転を適用する

## 10. 設定の永続化

| プロパティ（`AppSettings`） | 保存タイミング | 内容 |
|---------------------------|-------------|------|
| `CompareFormWidth` | `ResizeEnd` | ウィンドウ幅（px） |
| `CompareFormHeight` | `ResizeEnd` | ウィンドウ高さ（px） |
| `CompareSplitterDistance` | `SplitterMoved` | スプリッター位置（px） |
| `CompareListColumnWidths` | `ColumnWidthChanged` | リスト列幅の配列（px） |
| `ThumbnailRotations` | 回転操作時 | パスごとの回転回数 |

保存先: `%LOCALAPPDATA%\Nimono\settings.json`（メイン画面と共有）

---

## 11. ウィンドウ動作

| 項目 | 仕様 |
|------|------|
| 起動位置 | メイン画面の中央（`CenterParent`） |
| 最小サイズ | 600×400px |
| デフォルトサイズ | 900×700px（設定がない場合） |
| モード | モーダル（`ShowDialog`）。メイン画面は操作不可 |
| 終了時 | 読み込んだ Bitmap を破棄してリソースを解放 |

## 12. クラス構成

```
CompareForm
├── _group: ImageGroup          グループデータ（Paths / Similarities / Hashes / Embeddings）
├── _settings: AppSettings      設定（サイズ・スプリッター位置・列幅の読み書き）
├── _sizeCache: Dictionary      解像度文字列のキャッシュ
├── _mainSplitter: SplitContainer  上下分割
├── _leftView / _rightView      各 ZoomableImagePanel
├── _leftInfo / _rightInfo      各情報ラベル（RichTextBox、差分を赤字表示）
├── _fileList: ListView         ファイルリスト
├── _leftBitmap / _rightBitmap  表示中の Bitmap（Dispose 管理）
├── _suppressEvent: bool        プログラム選択時のイベント二重発火防止フラグ
├── _paths: List<string>        現在のファイルパス一覧（削除・基準変更で変動）
├── _similarities: Dictionary   現在の類似度マップ
└── _deletedPaths: List<string> 削除されたファイルパスの記録
```

---

## 13. 制約・注意事項

- 左パネルは常にグループ先頭（基準）画像で固定。メイン画面でサムネイルクリックにより基準が変更されている場合は、その時点の `group.Paths[0]` が使われる。
- 比較画面を開いたままメイン画面で基準変更を行っても、比較画面には反映されない（再度「比較」ボタンを押す必要がある）。
- 解像度列はファイルヘッダーの読み取りを伴うため、ファイル数が多いグループでは画面表示に若干時間がかかる場合がある。
