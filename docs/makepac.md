# MSIXパッケージの作成手順

NimonoプロジェクトのMSIXパッケージを再現可能に作成（ビルド・パッケージング）するためには、プロジェクトのルートディレクトリにある `publish.ps1` スクリプトを使用します。

このスクリプトは、ソースコードのビルド、マニフェストのコピー、パッケージ化、そして自己署名までを一貫して自動で行います。

## 作成方法

PowerShellを開き、プロジェクトのルートフォルダに移動して、以下のコマンドを実行します。

### 1. Microsoft Store 提出用（署名なし）

そのままスクリプトを実行します。生成されたMSIXファイルは `dist` フォルダ内に出力されます。

```powershell
.\publish.ps1
```

### 2. ローカルでのテスト用（自己署名あり）

ローカルPCにインストールしてテストしたい場合は、`-Sign` オプションを付けて実行します。これにより、テスト用の自己証明書が作成され、MSIXパッケージに署名が行われます。

```powershell
.\publish.ps1 -Sign
```

> **注意**: 署名付きでビルドした場合、インストール前に `dist` フォルダに生成される `.pfx` 証明書（例: `NimonoTest.pfx`）をローカルマシンの「信頼されたルート証明機関」にインストールする必要があります。

## スクリプトで行われている自動化の内容

スクリプトを実行すると、内部で以下の処理が自動で行われます。

1. `dotnet publish` による Release ビルド（`win-x64` 向け、自己完結型）。
2. アプリの構成ファイル (`AppxManifest.xml`) とアイコンなどの `Assets` フォルダをビルド出力先にコピー。
3. Windows SDK の `makeappx.exe` を使用してMSIXパッケージ化。
4. （`-Sign` 指定時）`signtool.exe` を使って自己署名を実行。
